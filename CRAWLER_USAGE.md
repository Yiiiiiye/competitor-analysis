# 爬虫功能使用说明

## 功能概述

本系统现已支持从社交媒体平台自动抓取竞品动态，支持的平台包括：
- 微博
- 抖音
- 小红书
- B站

## 部署步骤

### 1. 安装Python依赖

```bash
cd backend/crawler
pip install -r requirements.txt
```

### 2. 启动爬虫服务

```bash
cd backend/crawler
python server.py
```

爬虫服务将在 `http://127.0.0.1:5001` 启动

### 3. 启动后端服务

```bash
cd backend
npm install
npm run dev
```

后端服务将在 `http://localhost:5000` 启动

### 4. 启动前端服务

```bash
npm install
npm run dev
```

## 使用方法

### 配置竞品社交媒体账号

1. 进入"竞品管理"页面
2. 点击"添加竞品"或编辑现有竞品
3. 勾选"启用自动抓取"选项
4. 在"社交媒体账号配置"区域填写需要抓取的平台账号ID：
   - **微博**：填写微博账号的数字ID（如：1234567890）
   - **抖音**：填写抖音账号的数字ID（如：9876543210）
   - **小红书**：填写小红书账号的数字ID
   - **B站**：填写B站账号的数字ID（mid）
5. 勾选需要抓取的平台
6. 点击"确认"保存

### 抓取单个竞品

1. 在竞品列表中找到需要抓取的竞品
2. 点击该竞品行的"抓取"按钮
3. 等待抓取完成，系统会显示抓取结果

### 批量抓取所有竞品

1. 在竞品管理页面顶部
2. 点击"一键抓取所有竞品"按钮
3. 系统会自动抓取所有启用了自动抓取的竞品
4. 等待抓取完成，系统会显示总抓取数量

## 查看抓取结果

抓取完成后，可以：
1. 在"动态管理"页面查看所有抓取到的动态
2. 在"仪表盘"页面查看本周概览
3. 在"历史记录"页面查看历史周报

## 注意事项

### 账号ID获取方法

**微博账号ID：**
- 访问用户主页，查看URL中的数字部分
- 例如：https://weibo.com/u/1234567890，账号ID为 1234567890

**抖音账号ID：**
- 需要通过API或第三方工具获取
- 可以使用MediaCrawler项目获取账号信息

**小红书账号ID：**
- 访问用户主页，查看页面源代码中的user_id
- 或使用MediaCrawler项目获取

**B站账号ID：**
- 访问用户主页，查看URL中的数字部分
- 例如：https://space.bilibili.com/123456789，账号ID为 123456789

### 抓取限制

- 每次抓取每个平台最多获取10条最新动态
- 系统会自动去重，避免重复保存相同的动态
- 抓取频率建议不要过于频繁，避免被平台限制

### 常见问题

**Q: 抓取失败怎么办？**
A: 请检查：
1. 爬虫服务是否正常运行
2. 账号ID是否正确
3. 网络连接是否正常
4. 目标平台是否有反爬限制

**Q: 如何获取账号ID？**
A: 可以使用MediaCrawler项目（https://github.com/NanmiCoder/MediaCrawler）来获取账号信息。

**Q: 抓取的数据在哪里？**
A: 抓取的数据会自动保存到"动态管理"中，可以查看、编辑和删除。

**Q: 可以自定义抓取数量吗？**
A: 目前默认每次抓取10条，可以在 `backend/crawler/crawler.py` 中修改 `limit` 参数。

## 技术架构

```
前端 (Vue 3)
    ↓ HTTP请求
后端 (Express.js)
    ↓ HTTP请求
爬虫服务 (Flask + Python)
    ↓ 爬取
社交媒体平台 (微博、抖音、小红书、B站)
```

## API接口

### 后端接口

- `POST /api/crawler/fetch/:competitorId` - 抓取单个竞品
- `POST /api/crawler/fetch-all` - 批量抓取所有竞品
- `GET /api/crawler/status` - 检查爬虫服务状态

### 爬虫服务接口

- `POST /api/crawl` - 执行爬取任务
- `GET /api/health` - 健康检查
