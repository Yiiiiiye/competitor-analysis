# 二、产品概述

## 2.1 产品定位

轻量型竞品动态聚合工具，无需复杂开发，通过低代码/现成工具组合，实现竞品多渠道动态（官网公告、小红书、公众号）的自动抓取、结构化整理，最终输出“竞品新增功能→改动点→对自身的启示”的一页式周报，降低竞品监测的时间成本，提升决策效率。

## 2.2 核心价值

- 效率提升：将原本“每天30分钟+手动检索、整理”的工作，压缩至5分钟，减少无效操作

- 信息聚合：打破多渠道壁垒，无需分别打开官网、小红书、公众号，一站式获取核心竞品动态

- 结构化输出：自动提炼关键信息，按固定模板整理，避免信息杂乱，直接用于决策参考或汇报

- 轻量落地：无需专业开发能力，依托现成工具快速搭建，降低项目启动成本，可快速验证价值

## 2.3 解决痛点

- 痛点1：竞品动态分散在官网、小红书、公众号等多个渠道，手动检索繁琐，易遗漏关键信息

- 痛点2：手动整理竞品动态耗时久，且输出内容杂乱，难以快速提炼核心改动与启示

- 痛点3：缺乏固定模板，每次整理周报需重新排版，效率低下

- 痛点4：无法快速同步竞品核心动向，导致决策滞后或错失市场机会

# 三、核心需求与功能需求

## 3.1 核心需求

用户可通过工具快速聚合核心竞品的多渠道动态，自动整理成结构化一页式周报，实现“每天5分钟掌握竞品动向”，获取可落地的竞品启示。

## 3.2 功能需求（按模块划分，MVP版本优先实现核心功能）

### 3.2.1 竞品管理模块

- 功能1：竞品添加/编辑/删除
        

    - 支持添加3-5个核心竞品（限制数量，保证轻量性），填写竞品名称、核心业务、对应监测渠道（官网地址、公众号名称、小红书官方账号）

    - 支持编辑竞品信息（如渠道更新、竞品业务调整），支持删除无需继续监测的竞品

    - 默认保存竞品列表，下次进入无需重新添加

- 功能2：竞品优先级设置


    - 支持给竞品标记优先级（高/中/低），周报中按优先级排序展示，优先突出高优先级竞品动态

### 3.2.2 渠道聚合与动态抓取模块

- 功能1：多渠道支持
        

    - 核心支持3个渠道：竞品官网公告、竞品官方公众号、竞品小红书官方账号

    - 无需开发爬虫，依托RSS订阅、公众号推送、小红书轻量监测工具（如微小宝、新榜、RSSHub）实现动态抓取

- 功能2：自动抓取与更新
        

    - 抓取频率：每天1次（固定时间，如每日上午9点），确保动态时效性

    - 抓取内容：仅抓取竞品功能更新、版本迭代、策略调整相关内容，排除日常推文、活动促销、品牌广告等无关信息

    - 异常提醒：若某渠道抓取失败（如链接失效、账号异常），给出简单提醒，用户可手动更新渠道信息

- 功能3：动态筛选
        

    - 支持按竞品、渠道、时间范围（近1天、近7天）筛选动态，快速定位所需信息

### 3.2.3 AI结构化整理模块

- 功能1：自动提炼关键信息
        

    - 将抓取到的竞品动态原文（官网公告、公众号推文、小红书笔记），通过AI（豆包/GPT）自动提炼核心信息，输出固定结构：竞品名称、渠道、发布时间、新增功能、关键改动、对自身的启示

    - “对自身的启示”需贴合用户业务，突出可落地性，避免空话、套话

- 功能2：人工编辑调整
        

    - AI整理后的内容，支持用户手动编辑、修改，确保信息准确性和针对性（如补充自身业务相关的启示）

### 3.2.4 周报生成与输出模块

- 功能1：固定模板生成

    - 周报模板固定为一页式，结构如下：

    - 标题：竞品动态轻量周报｜YYYY-MM-DD（周度）

    - 本周概览（3句话）：新增功能总数、重大改动总数、关键机会/风险1个

    - 单竞品拆解（按优先级排序）：每个竞品对应“新增功能→关键改动→对我的启示”三段式

    - 本周核心结论（仅1条）：提炼最有价值的决策参考，如“竞品均上线XX功能，建议我方优先排期验证”

    - 系统自动将AI整理后的内容填充至模板，无需用户手动排版

- 功能2：周报输出
        

    - 支持导出为PDF、Word格式，方便用户保存、分享或汇报

    - 支持直接复制周报内容，粘贴至企业微信、钉钉等办公工具

- 功能3：周报历史留存
        

    - 自动留存历史周报，支持按时间范围检索、查看，方便用户对比竞品长期动态趋势

### 3.2.5 设置管理模块

- 功能1：抓取时间设置


    - 支持用户调整动态抓取时间（默认每日9点），适配用户工作习惯

- 功能2：模板自定义（可选，MVP版本可暂不实现）
        

    - 后续可支持用户自定义周报模板，调整模块顺序、补充自定义字段

- 功能3：账号与数据备份
        

    - 支持简单账号登录（如微信登录），同步竞品列表、历史周报数据，避免数据丢失

# 四、非功能需求

## 4.1 易用性

- 操作流程简单，新用户10分钟内可完成竞品添加、渠道配置，上手无门槛

- 界面简洁，无冗余功能，核心操作（查看动态、生成周报）一键可达

- AI整理速度快，单条动态整理耗时≤10秒，周报生成耗时≤30秒

## 4.2 稳定性

- 动态抓取成功率≥90%，若抓取失败，给出清晰提醒及解决建议（如检查渠道链接）

- 系统无频繁崩溃，单次使用过程中无卡顿（响应时间≤2秒）

- 历史周报数据留存稳定，无丢失、错乱情况

## 4.3 准确性

- AI提炼的核心信息（新增功能、改动点）准确率≥85%，允许用户手动修正

- 抓取的动态无重复、无无关内容（如活动促销、无关推文）

## 4.4 轻量性

- 无需下载重型客户端，优先采用网页端、小程序或Notion/飞书模板形式，降低用户使用成本

- 无需专业技术能力，全程可视化操作，无需编写代码、配置复杂参数

# 五、用户流程

## 5.1 核心用户流程（MVP版本）

1. 初始化配置：用户注册/登录 → 添加3-5个核心竞品 → 配置每个竞品的监测渠道（官网、公众号、小红书）

2. 动态获取：系统每日自动抓取各渠道竞品动态 → 过滤无关内容 → AI自动整理成结构化信息

3. 周报生成：用户查看AI整理后的动态，手动微调 → 一键生成一页式周报

4. 周报使用：用户导出周报（PDF/Word） → 分享/汇报 → 查看历史周报（如需）

## 5.2 关键流程说明

整个流程核心耗时控制在每天5分钟内：系统自动抓取（无需用户操作），用户仅需花2分钟查看、微调AI整理内容，3分钟生成、导出周报。

# 九、风险与应对方案

|潜在风险|应对方案|
|---|---|
|渠道抓取失败（如官网改版、公众号屏蔽、小红书账号变更）|1. 给出清晰的失败提醒，告知用户检查渠道信息；2. 提供手动添加动态的入口，避免遗漏关键信息；3. 定期优化抓取方式，适配渠道变化|
|AI整理内容不准确，无法满足用户需求|1. 优化AI提示词，提升提炼准确性；2. 强化人工编辑功能，允许用户自由修改内容；3. 收集用户反馈，持续迭代AI算法|
|用户觉得操作繁琐，无法实现“每天5分钟”的目标|1. 简化操作流程，减少不必要的步骤；2. 优化自动化程度，如自动填充常用配置、默认生成周报；3. 提供操作指南，帮助用户快速上手|
|数据丢失（如竞品列表、历史周报）|1. 支持账号登录同步数据，定期自动备份；2. 提供手动导出备份的功能，让用户自主保存关键数据|
